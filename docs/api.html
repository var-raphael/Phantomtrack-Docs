API Access

Programmatically access your analytics data from any platform

What is the PhantomTrack API?

The PhantomTrack API allows you to programmatically retrieve analytics data from your websites. Perfect for building custom dashboards, mobile apps, integrations, or automating reports.

Real-Time Data

Access up-to-date analytics instantly

Secure & Private

API keys with role-based access

Easy Integration

Simple REST API with JSON responses

Test Mode

Free testing with dummy data

Quick Start Guide

Get started with the PhantomTrack API in 3 simple steps:

1

Get Your API Keys

Navigate to your PhantomTrack Dashboard → Settings → API Keys. You'll find both test and production keys ready to use.

2

Make Your First Request

Use your test key to fetch analytics data. Test keys return dummy data and have no rate limits.

3

Go to Production

Once ready, switch to your production key to access real analytics data.

API Key Types

PhantomTrack provides two types of API keys for different use cases:

Test Keys (sk_test_...)

  • Returns realistic dummy data
  • No rate limits - unlimited requests
  • Perfect for development and testing
  • Safe to expose in public code
  • No usage tracking

Production Keys (sk_live_...)

  • Returns your real analytics data
  • Subject to plan-based rate limits
  • Tracks API usage towards monthly quota
  • Should be kept secret
  • Includes AI-generated insights

Security Warning: Never expose production API keys in client-side code, public repositories, or browser JavaScript. Always use them server-side. If exposed, regenerate immediately from your dashboard settings.

Available Data

Access comprehensive analytics through dedicated endpoints:

Core Metrics

  • /stats - Overall statistics (unique visitors, pageviews, live visitors, bounce rate)
  • /pages - Most visited pages with view counts and time spent
  • /referrers - Top traffic sources and referrers
  • /countries - Visitor breakdown by country

User Insights

  • /devices - Device type distribution (mobile, desktop, tablet)
  • /browsers - Browser usage statistics

Custom Events

  • /events - Your tracked custom events with counts

Account Info

  • /usage - Current API usage and remaining quota
  • /all - Everything in one request (perfect for dashboards)

See the complete Endpoints Documentation for detailed information on each endpoint.

Rate Limits & Quotas

Understanding API limits:

IP-Based Limits (All Keys)

  • 60 requests/minute per IP address
  • 10 requests/second per website (burst protection)

Monthly Quotas (Production Keys Only)

  • Free Plan: 10,000 requests/month
  • Pro Plan: 30,000 requests/month ($3/mo)
  • Premium Plan: 60,000 requests/month ($5/mo)
  • Enterprise Plan: 100,000 requests/month ($8/mo)
  • Lifetime Plan: Unlimited requests ($20 one-time)

Test Keys: Test keys have NO rate limits and don't count towards your quota. Use them freely!

Integration Examples

JavaScript / Node.js

const fetch = require('node-fetch');

const API_KEY = process.env.PHANTOMTRACK_API_KEY;

async function getAnalytics() {
  const response = await fetch(
    'https://phantomtrack.42web.io/api/analytics?endpoint=all&period=7d',
    {
      headers: { 'X-API-KEY': API_KEY }
    }
  );
  
  const data = await response.json();
  return data;
}

getAnalytics()
  .then(data => console.log(data))
  .catch(err => console.error(err));

Python

import requests
import os

API_KEY = os.getenv('PHANTOMTRACK_API_KEY')
BASE_URL = 'https://phantomtrack.42web.io/api/analytics'

def get_analytics():
    headers = {'X-API-KEY': API_KEY}
    params = {'endpoint': 'all', 'period': '7d'}
    
    response = requests.get(BASE_URL, headers=headers, params=params)
    return response.json()

if __name__ == '__main__':
    data = get_analytics()
    print(data)

Common Questions

Q: Can I use the API from client-side JavaScript?

A: Yes, but only with test keys. Never expose production keys in browser JavaScript. For production data, use server-side API calls.

Q: How often can I make requests?

A: You can make up to 10 requests per second and 60 requests per minute per IP. Monthly quotas depend on your plan.

Q: Does the API cost extra?

A: No! API access is included with all plans. You're only limited by your plan's monthly request quota.

Q: What if I hit my monthly limit?

A: The API will return a 429 error. You can upgrade your plan or wait until next month for the quota to reset.

Q: Can I regenerate my API keys?

A: Yes! Go to Dashboard → Settings → API Keys and click "Regenerate". The old key will be invalidated immediately.

Q: Are there webhook integrations?

A: Not yet, but they're on our roadmap! For now, you can poll the API or use scheduled tasks.

Q: Is there an API for tracking events?

A: The API is read-only for retrieving data. To track events, use the phantom.track() JavaScript function on your website.

Security Best Practices

Keep your API integrations secure:

DO:

  • Store API keys in environment variables
  • Use server-side code for production keys
  • Rotate keys periodically for enhanced security
  • Monitor your API usage in the dashboard
  • Use test keys for development and testing
  • Implement proper error handling
  • Use HTTPS for all API requests

DON'T:

  • Commit API keys to version control (GitHub, GitLab)
  • Expose production keys in client-side JavaScript
  • Share API keys publicly or in screenshots
  • Hardcode keys in your source code
  • Use the same key across multiple projects

Key Exposed? If you accidentally exposed your production key, regenerate it immediately from Settings → API Keys. The old key will stop working instantly.

Error Handling

The API returns standard HTTP status codes:

Success Codes

  • 200 OK - Request successful

Error Codes

  • 401 Unauthorized - Missing or invalid API key
  • 403 Forbidden - API key format invalid or key not found
  • 404 Not Found - Invalid endpoint
  • 429 Too Many Requests - Rate limit or quota exceeded
  • 500 Internal Server Error - Server error (contact support)

Handling Errors in Code

async function getAnalyticsWithRetry() {
  try {
    const response = await fetch(API_URL, {
      headers: { 'X-API-KEY': API_KEY }
    });
    
    if (!response.ok) {
      const error = await response.json();
      
      if (response.status === 429) {
        console.error('Rate limit exceeded:', error.message);
        // Wait and retry or notify user to upgrade
      } else if (response.status === 401) {
        console.error('Invalid API key');
      } else {
        console.error('API error:', error);
      }
      
      return null;
    }
    
    return await response.json();
  } catch (error) {
    console.error('Network error:', error);
    return null;
  }
}

Pro Tips

Optimize API Usage

  • Use the /all endpoint to get everything in one request
  • Cache responses to reduce API calls
  • Use appropriate date ranges - don't fetch more data than needed
  • Implement exponential backoff for rate limit errors

Better Performance

  • Make parallel requests for different endpoints when needed
  • Use pagination parameters when available (limit parameter)
  • Schedule heavy API calls during off-peak hours
  • Monitor response times and optimize accordingly

Development Workflow

  • Always test with test keys first
  • Build with dummy data before going to production
  • Use environment variables for easy key switching
  • Log API usage during development to estimate production needs

Going to Production

  • Double-check all test keys are replaced with production keys
  • Ensure production keys are never exposed client-side
  • Set up monitoring for API errors and rate limits
  • Have a plan for handling quota exhaustion
