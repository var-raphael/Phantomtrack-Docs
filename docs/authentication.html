API Authentication

Secure your API access with test and production keys

Authentication Overview

PhantomTrack uses API keys to authenticate requests. Every API request must include a valid API key to access your analytics data.

Two Types of API Keys:

  • Test Keys - For development and testing with dummy data
  • Production Keys - For live applications with real analytics data

Test API Keys

TEST Test keys return dummy data and have no rate limits. Perfect for development!

sk_test_abcdef1234567890ghijklmnop

Test Key Features:

  • ✓ Returns realistic dummy analytics data
  • ✓ No rate limits - unlimited requests
  • ✓ Safe to use in public repositories
  • ✓ Perfect for development and testing
  • ✓ Can be regenerated anytime

Best Practice: Always use test keys during development. Switch to production keys only when you're ready to go live.

Production API Keys

PRODUCTION Production keys access your real analytics data and are subject to plan limits.

sk_live_zyxwvu9876543210nmlkjihgfe

Production Key Features:

  • ✓ Returns real analytics data from your website
  • ✓ Subject to plan-based rate limits
  • ✓ Tracks API usage towards monthly quota
  • ✓ Should be kept secret and secure
  • ✓ Can be regenerated if compromised

Security Warning: Never expose production API keys in client-side code, public repositories, or browser JavaScript. If your key is exposed, regenerate it immediately from your dashboard settings.

Test vs Production Keys

Understanding the differences between test and production keys:

Feature Test Key Production Key
Data Type Dummy/Sample Data Real Analytics Data
Rate Limits Unlimited Plan-Based
Usage Tracking Not Counted Counted Towards Quota
Security Level Low (Public OK) High (Keep Secret)
Best For Development & Testing Production Applications
Key Format sk_test_... sk_live_...

How to Authenticate

Include your API key in requests using one of these methods:

Method 1: X-API-KEY Header (Recommended)

curl -H 'X-API-KEY: sk_test_your_key_here' \
  https://phantomtrack.42web.io/api/analytics?endpoint=stats

Method 2: Authorization Bearer Header

curl -H 'Authorization: Bearer sk_test_your_key_here' \
  https://phantomtrack.42web.io/api/analytics?endpoint=stats

Method 3: Query Parameter

curl 'https://phantomtrack.42web.io/api/analytics?endpoint=stats&api_key=sk_test_your_key_here'

For production keys, always use headers (Method 1 or 2) to avoid exposing your key in URLs or logs.

Code Examples

JavaScript (Fetch API)

fetch('https://phantomtrack.42web.io/api/analytics?endpoint=stats', {
  method: 'GET',
  headers: {
    'X-API-KEY': 'sk_test_your_key_here'
  }
})
.then(response => response.json())
.then(data => {
  console.log('Analytics:', data);
})
.catch(error => {
  console.error('Error:', error);
});

Python (Requests)

import requests

headers = {
    'X-API-KEY': 'sk_test_your_key_here'
}

response = requests.get(
    'https://phantomtrack.42web.io/api/analytics',
    params={'endpoint': 'stats'},
    headers=headers
)

data = response.json()
print('Analytics:', data)

PHP (cURL)

 'https://phantomtrack.42web.io/api/analytics?endpoint=stats',
    CURLOPT_RETURNTRANSFER => true,
    CURLOPT_HTTPHEADER => [
        'X-API-KEY: sk_test_your_key_here'
    ]
]);

$data = json_decode($response, true);

print_r($data);
?>

Error Responses

Common authentication error responses:

Missing API Key (401 Unauthorized)

{
  "success": false,
  "error": "Unauthorized",
  "message": "API key is required. Provide via X-API-KEY header, Authorization: Bearer header, or api_key parameter."
}

Invalid API Key (403 Forbidden)

{
  "success": false,
  "error": "Forbidden",
  "message": "Invalid API key format"
}

Rate Limit Exceeded (429 Too Many Requests)

{
  "success": false,
  "error": "API Limit Reached",
  "message": "Monthly API limit of 10000 requests reached for free plan.",
  "current_usage": 10000,
  "limit": 10000,
  "plan": "free",
  "upgrade_url": "https://phantomtrack.42web.io/pricing"
}

Regenerating API Keys

If your production API key is exposed or compromised:

  1. Go to your PhantomTrack Dashboard
  2. Navigate to Settings → API Keys
  3. Click Regenerate Production Key
  4. Update your application with the new key
  5. The old key will be invalidated immediately

Important: Regenerating your production key will immediately invalidate the old key. Make sure to update all applications using the old key to avoid service interruption.

Security Best Practices

✓ DO:

  • Use test keys for all development and testing
  • Keep production keys in environment variables
  • Use server-side code for production API calls
  • Regenerate keys if they're exposed
  • Monitor your API usage regularly
  • Use HTTPS for all API requests

✗ DON'T:

  • Commit production keys to version control
  • Expose production keys in client-side JavaScript
  • Share your production keys publicly
  • Use production keys in public demos or documentation
  • Hard-code keys directly in your source code
  • Log production keys in error messages
