Custom Events Tracking

Track button clicks, form submissions, and any user interaction on your website.

What are Custom Events?

Custom events let you track specific user actions beyond basic page views. Monitor button clicks, video plays, form submissions, downloads, or any interaction you care about.

What you can track:

  • Button clicks (CTA buttons, signup buttons, etc.)
  • Form submissions (contact forms, newsletter signups)
  • File downloads (PDFs, images, documents)
  • Video interactions (play, pause, complete)
  • E-commerce actions (add to cart, checkout started)
  • Any custom user interaction

Basic Usage

PhantomTrack gives you a global window.phantom.track() function. Use it anywhere in your JavaScript:

phantom.track('button_clicked');

You can also use the alias phantom.event() - they do the same thing:

phantom.event('button_clicked');

Simple Button Click Example

Track when a user clicks a button:

<button onclick="phantom.track('signup_clicked')">
  Sign Up
</button>

Adding Properties to Events

Track additional data with your events using properties. This is useful for context like product names, prices, or user selections.

phantom.track('product_viewed', {
  product_name: 'Blue T-Shirt',
  price: 29.99,
  category: 'clothing'
});

Properties Limit: Properties must be under 10KB (10,000 characters) total when converted to JSON.

Real-World Examples

Form Submission

Track when users submit a contact form:

<form onsubmit="phantom.track('contact_form_submitted'); return true;">
  <input type="email" name="email" placeholder="Your email" />
  <button type="submit">Submit</button>
</form>

File Download

Track PDF or file downloads:

<a href="/files/pricing.pdf" 
   onclick="phantom.track('pdf_downloaded', { file: 'pricing.pdf' })">
  Download Pricing PDF
</a>

Add to Cart

Track e-commerce actions:

<button onclick="addToCart()">Add to Cart</button>

<script>
function addToCart() {
  // Your cart logic here
  
  phantom.track('add_to_cart', {
    product_id: 'prod_123',
    product_name: 'Wireless Headphones',
    price: 79.99,
    quantity: 1
  });
}
</script>

Video Play Tracking

Track when users interact with videos:

<video id="myVideo" controls>
  <source src="video.mp4" type="video/mp4">
</video>

<script>
const video = document.getElementById('myVideo');

video.addEventListener('play', function() {
  phantom.track('video_played', {
    video_name: 'Product Demo'
  });
});

video.addEventListener('ended', function() {
  phantom.track('video_completed', {
    video_name: 'Product Demo'
  });
});
</script>

Framework-Specific Examples

React

Track events in React components:

import React from 'react';

function SignupButton() {
  const handleClick = () => {
    // Your signup logic
    
    window.phantom.track('signup_clicked', {
      button_location: 'header'
    });
  };
  
  return (
    <button onClick={handleClick}>
      Sign Up
    </button>
  );
}

export default SignupButton;

Vue.js

Track events in Vue components:

<template>
  <button @click="trackSignup">
    Sign Up
  </button>
</template>

<script>
export default {
  methods: {
    trackSignup() {
      // Your signup logic
      
      window.phantom.track('signup_clicked', {
        component: 'SignupButton'
      });
    }
  }
}
</script>

jQuery

Track events with jQuery:

$(document).ready(function() {
  $('#signup-btn').click(function() {
    phantom.track('signup_clicked', {
      source: 'homepage'
    });
  });
  
  $('#newsletter-form').submit(function() {
    phantom.track('newsletter_signup');
  });
});

Best Practices

✓ DO:

  • Use descriptive event names: signup_clicked, pdf_downloaded
  • Use underscores for multi-word events: add_to_cart
  • Keep event names under 100 characters
  • Add properties for context: product names, prices, categories
  • Test events in browser console first

✗ DON'T:

  • Include personal data (emails, names, phone numbers)
  • Use spaces in event names: use button_clicked not button clicked
  • Send properties larger than 10KB
  • Track every tiny interaction (focus on important actions)

Testing Your Events

Open your browser console (F12) and test tracking manually:

phantom.track('test_event', {
  message: 'This is a test'
});

Then check your PhantomTrack dashboard to see if the event appears in your custom events list.

Events appear in your dashboard within seconds. Refresh to see the latest data.
